Resumen del proyecto

El sistema fue diseñado para monitorear variables ambientales críticas en un invernadero usando una arquitectura IoT. El Arduino Uno R4 WiFi funciona como unidad central y se conecta a sensores de temperatura, humedad, gases y movimiento. Además, incorpora un reloj en tiempo real para registrar fecha y hora, una pantalla LCD para visualizar datos en sitio, una microSD para almacenar historial y conectividad WiFi para mostrar información en red local y en la nube.

🎯

¿Qué problema resuelve?

Situación inicial

En muchos cultivos o invernaderos pequeños, el monitoreo todavía depende de revisiones manuales. Eso hace que cambios de temperatura, humedad o calidad del aire pasen desapercibidos por un tiempo, afectando el ambiente del cultivo y dificultando llevar un control histórico confiable.

Solución propuesta

Este sistema automatiza la lectura de variables ambientales, registra la información con fecha y hora, la muestra localmente y la comparte por WiFi. Así se reduce la dependencia de supervisión constante y se facilita el análisis posterior con datos reales.

Metodología

El desarrollo del proyecto se llevó a cabo siguiendo un enfoque práctico basado en la implementación de un sistema real, utilizando una metodología por etapas que permitió avanzar de forma organizada desde el diseño hasta la validación final.

1

Diseño del sistema

Primero se definió la arquitectura general del proyecto. En esta parte se escogieron los componentes principales, incluyendo el Arduino Uno R4 WiFi como unidad central, los sensores DHT22, MQ-135 y PIR, además del RTC, la pantalla LCD y el módulo microSD. También se establecieron las conexiones necesarias entre los dispositivos.

2

Implementación del hardware

Luego se realizó el montaje físico en protoboard. En esta etapa se conectaron todos los componentes respetando sus requerimientos eléctricos y de comunicación, organizando la distribución para facilitar el acceso y evitar interferencias.

3

Desarrollo del software

Después se programó el sistema en Arduino IDE usando C++. Se implementó la lectura de sensores, el manejo del RTC, el guardado de datos en microSD, la visualización en LCD y la conexión WiFi para servidor web local y envío a ThingSpeak.

4

Pruebas y validación

Finalmente se evaluó el funcionamiento del sistema completo, verificando la lectura de los sensores, la estabilidad de la conexión, el almacenamiento de datos, la actualización de la interfaz web y la respuesta del sensor PIR ante movimiento.

¿Cómo funciona el código?

Aunque el proyecto tiene varias librerías y módulos, la lógica principal sigue una secuencia sencilla.

1

Leer sensores

El programa toma los datos del DHT22, MQ-135 y PIR en intervalos definidos.

2

Procesar información

El Arduino organiza los valores, calcula diferencias y prepara los datos para mostrar o guardar.

3

Agregar fecha y hora

El RTC DS3231 aporta el tiempo exacto para registrar cada lectura.

4

Guardar y mostrar

Los datos se escriben en microSD y también se visualizan localmente en la pantalla LCD.

5

Enviar por WiFi

Finalmente, el sistema publica la información en web local y la envía a ThingSpeak.

⚠️

Limitaciones del sistema

Precisión del MQ-135

El sensor MQ-135 sirve bien para comparar cambios en la calidad del aire, pero no entrega valores absolutos muy precisos sin una calibración más estricta.

Dependencia de red

Las funciones web y el envío a la nube dependen de que la red WiFi esté disponible y estable en el momento de uso.

Espacio de visualización

La LCD 16x2 es útil para ver datos rápidos, pero tiene espacio muy limitado para mostrar información más detallada.

Escalabilidad

El Arduino Uno R4 WiFi soporta bien este proyecto, aunque agregar muchos más sensores o análisis complejos puede requerir una plataforma más robusta.

Estructura del sistema en conjunto

Orden lógico de funcionamiento

  1. Los sensores capturan datos del entorno.
  2. El Arduino procesa la información recibida.
  3. El RTC agrega fecha y hora a cada lectura.
  4. La LCD muestra datos en tiempo real.
  5. La microSD guarda registros en formato CSV.
  6. El WiFi publica la información en web local y ThingSpeak.

Conexiones más importantes

  • DHT22 → pin D2
  • PIR → pin D3
  • MQ-135 → pin A0
  • microSD → CS en D4
  • RTC y LCD → bus I2C
  • WiFi → integrado en Arduino Uno R4 WiFi
Montaje recomendado: sensores a un lado del protoboard, módulos de visualización y almacenamiento al otro, con alimentación común en 5V y GND para mantener el cableado más limpio y entendible.

Preguntas rápidas

¿Para qué sirve este proyecto en la vida real?

Sirve para vigilar condiciones importantes en un invernadero sin depender de revisión manual constante. Esto ayuda a tomar decisiones más rápidas y a registrar el comportamiento del ambiente con datos reales.

¿Por qué usar varios módulos y no solo sensores?

Porque medir no es suficiente. También se necesita mostrar, guardar, fechar y compartir la información. Por eso se integran pantalla, reloj en tiempo real, microSD y conectividad WiFi.
